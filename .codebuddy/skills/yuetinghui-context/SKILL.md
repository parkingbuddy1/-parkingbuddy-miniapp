## 二、粤停汇核心商业模式

### 2.1 业务本质

**车位二级分销 + 产品再设计**——本质是"资源、运营和风控"的复合生意，难点不在卖，而在"拿、控、稳"。

### 2.2 三步模式

1. **批发拿位**：从写字楼/科创园区的物业或开发商手中批发租赁部分车位（如 1000 个车位中租 100 个），利用批发优势压低租赁价格（如正常月卡 500 元/张，批发价 300 元/张）
2. **零售转租**：将车位通过比物业低一点的零售价，转租给写字楼/园区租户、员工，赚取差价
3. **产品化扩展**：停车场数量多起来后，推出多种卡型：
   - 普通月卡（写字楼/园区）
   - 停充一体月卡（加装充电桩）
   - 全城通卡（A 写字楼月卡用户加 20% 租金，实现全部停车场通停）
   - 日卡（限停一天）
   - 小时卡（限停 XX 小时）
   - 周末卡（周末通停）
   - 夜间卡（夜间通停）
   - 一周 X 次卡（一周限停 X 次，每次 X 分钟）

### 2.3 业务推广模式

1. 写字楼内上门推销
2. 物业内部推荐
3. 车辆预留电话进行电话销售
4. 停车场内部墙面广告销售

### 2.4 模式核心判断标准

1. 能否持续拿到"低于市场价 30% 以上"的车位 → 如果不能，利润会被吃掉
2. 能否把车位"产品化" → 不是简单月卡，而是分时、分级、跨场景

## 三、核心难点与解决方案（OpenAI 讨论成果）

### 难点 1：拿不到真正"有利润空间"的车位

**问题**：很多停车场看似有空位，但高峰时段（工作日白天）是满的，只有夜间或周末空置，或物业根本不愿意让渡定价权。结果：批发来的不是有效库存，而是"错时库存"。

**解决方案**：
- 不要只谈"总车位数"，要拆成：工作日白天占用率、夜间占用率、周末占用率
- 只拿"结构性空置"的车位（如夜间空、白天满的住宅/办公错配区）
- 争取分时定价权（如只买夜间资源）

### 难点 2：物业/开发商的博弈心理

**问题**：物业担心低价卖拉低原有月卡体系、抢走优质客户、后期做大后反客为主。

**解决方案**：从"租车位的人"变成"帮他们提升收益的人"
- 做"增量协议"：只销售未使用车位，或限定某些时段/楼栋
- 做"保底+分成"：保证最低收入 + 超额分成
- 给物业"控制权"：定价区间他们有否决权，品牌不对外强调你（白标）

### 难点 3：用户信任与确定性

**问题**：用户怕今天能停明天没位、被物业拦、权益不清晰。

**解决方案**：
- 电子化车牌识别系统打通
- 权益系统（类似 SaaS 后台）
- 明确是否保证车位 vs 仅"有概率停车"
- 价格分层：保证车位（贵）vs 非保证（便宜）

### 难点 4：跨停车场"通停"复杂度

**问题**：各停车场规则不同、高峰期冲突、结算复杂。

**解决方案**：初期分级推进
- 同区域通停（2-3 个场）
- 限时通停（如每天最多停 8 小时）
- 次数通卡（如每月 30 次）
- 本质：把风险锁死在规则里

### 难点 5：销售成本远高于预期

**问题**：客单价低（几百/月）、转化周期长、B 端+个人混合客户。

**解决方案**：
- B 端批量签约（企业）→ 一个公司 = 几十张卡
- 物业绑定销售 → 把他们变渠道（分佣）

### 难点 6：现金流风险

**问题**：先付物业（批发），再慢慢卖用户，卖不掉直接亏。

**解决方案**：
- 前期小批量试点（10-30 个车位）
- 预售制（月卡先卖再拿车位）
- 浮动租赁（按实际售出结算）

## 四、谈车场时必须盯死的数据

### 第一优先级（决定赚不赚钱）

| 数据 | 说明 |
|------|------|
| **分时段占用率** | 核心中的核心。工作日白天(8:00-18:00)、夜间(18:00-次日)、周末分别的占用率，直接决定有没有"可卖库存" |
| **月卡价格体系** | 当前月卡价格、是否分档（固定/非固定）、是否已满 → 判断有没有定价空间 |
| **临停收入 vs 月卡收入占比** | 临停收入高 → 流动性强，不适合；月卡占比高 → 更稳定，更适合 |

### 第二优先级（决定规模和效率）

| 数据 | 说明 |
|------|------|
| **总车位数 & 可释放比例** | 关键不是总量，而是能不能拿到"连续的一批" |
| **单车位年产值（ARPU）** | 年收入 ÷ 总车位数，判断是不是"高价值场" |
| **客户结构** | 企业客户多 vs 个人多，是否集中 → 决定销售难度 |

### 第三优先级（隐藏风险）

| 数据 | 说明 |
|------|------|
| **产权 & 决策权** | 谁能签合同（物业？开发商？业委会？） |
| **系统能力** | 是否有车牌识别，是否支持对接 → 没系统会被运营拖死 |

## 七、YTHPCS V3.0 详细内容

### 7.1 编制背景

停车行业问题：停车位利用率低（尤其住宅小区白天闲置）、车主停车信息分散、停车管理依赖人工经验、不同停车场缺乏统一信用机制、物业和业委会担忧共享停车风险。

**核心理念**：不是城市没有停车位，而是停车位没有被科学调度。

### 7.2 建设目标

1. 建立城市停车信用体系（一车一档、一人一档）
2. 提升停车资源利用率（AI 调度实现资源共享）
3. 降低物业管理成本（系统自动完成预约审核、违规管理）
4. 保障社区安全（实名预约、信用审核、车辆轨迹、违规预警）
5. 构建停车信用联盟（跨停车场信用互认）

### 7.3 五项核心理念

1. **信用优先**：停车资格建立在信用基础之上，信用越高服务越优
2. **安全优先**：任何共享停车行为不得突破社区安全底线
3. **AI 优先**：优先通过 AI 自动识别停车行为，减少人工干预
4. **数据驱动**：所有停车行为均形成数据记录
5. **共建共享**：物业、业委会、车主三方协同

### 7.4 信用对象（四类主体）

| 主体 | 评价内容 |
|------|----------|
| **车主信用（Driver Credit）** | 预约履约率、文明停车、超时记录、投诉记录、信用积分、黑名单状态 |
| **车辆信用（Vehicle Credit）** | 车辆基本信息、停车历史、违规记录、滞留记录、AI 风险标签 |
| **停车场信用（Parking Credit）** | 开放共享比例、用户满意度、收益稳定性、违规处理效率、数据完整性 |
| **物业信用（Property Credit）** | 服务响应速度、投诉处理率、数据上传率、共享开放率、合作稳定性 |

### 7.5 信用生命周期

```
实名认证 → 首次预约 → 信用建立 → 持续停车行为 → AI 实时评分 → 信用升级/下降 → 信用恢复/黑名单 → 长期信用档案
```

### 7.6 YTHPCS 定位调整

YTHPCS 定义为 YTHOS 中负责"信用与风控"的标准，另设 YTHOMS（停车运营标准）负责运营流程，职责更清晰。

### 7.7 正文编写标准

从大纲式输出升级为企业标准正文格式（如 1.2.1、1.2.2 等小节），每节 3-5 页 Word，整本目标 200-300 页。
